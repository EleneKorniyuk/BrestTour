# Экскурсионный лендинг «БрестТур» (BrestTour)

Адаптивный одностраничный сайт (landing page) для заказа экскурсий по направлению **«Тысячелетний Брест и Брестская крепость»** для платформы Ekskursii.by.

## Стек технологий

*   **HTML5** (Семантическая вёрстка)
*   **CSS3** (Компонентный подход, Flexbox, Grid, CSS Variables)
*   **Vanilla JavaScript** (ES6+, без сторонних тяжелых фреймворков)
*   **БЭМ** (Методология именования классов)
*   **Flatpickr** (Легковесная библиотека календаря для выбора дат)
*   **Font Awesome v6.4.0** (Иконочный шрифт)
*   **Google Fonts (Inter)** (Шрифт)

## Структура проекта и архитектура стилей

В проекте используется модульная структура стилей. Все CSS-файлы разделены по логическим компонентам и утилитам, после чего импортируются в главный файл стилей.

```text
BrestTour/
├── assets/                  # Графические ресурсы (логотипы, иконки, изображения)
├── scripts/
│   └── main.js              # Основной скрипт инициализации и интерактивности
├── styles/
│   ├── components/          # Стили независимых блоков
│   │   ├── header/          # Модульные стили шапки сайта
│   │   │   ├── header-core.css      # Позиционирование и контейнер хедера
│   │   │   ├── header-nav.css       # Навигация (десктопная версия)
│   │   │   ├── header-dropdowns.css # Выпадающие списки (валюта, язык, контакты)
│   │   │   ├── header-burger.css    # Анимация кнопки мобильного меню
│   │   │   └── header-mobile.css    # Стили раскрытого мобильного меню
│   │   ├── buttons.css      # Кнопки (Бронировать, Поиск, Расписание)
│   │   ├── footer.css       # Подвал сайта (контакты, копирайт)
│   │   ├── hero.css         # Главный экран с фоновым видео/изображением
│   │   ├── reviews.css      # Блок отзывов и элементы слайдера
│   │   └── schedule.css     # Форма поиска и сетка карточек экскурсий
│   ├── utils/               # Вспомогательные стили
│   │   ├── animations.css   # Ключевые кадры (@keyframes) для fadeInUp, pulse и др.
│   │   ├── flatpickr.css    # Кастомизация темы календаря
│   │   └── responsive.css   # Медиа-запросы (@media) для Mobile и Tablet
│   ├── base.css             # Сброс стилей (reset), шрифты и базовые теги
│   ├── variables.css        # CSS-переменные (цветовая палитра)
│   └── main.css             # Главный файл стилей (точка сборки @import)
└── index.html               # Основная HTML-разметка страницы
```

## Адаптивность

* **Desktop** --- ≥1024px --- Полная версия с горизонтальной навигацией
* **Tablet** --- 768–1023px --- Адаптивная сетка, бургер-меню
* **Mobile** --- 360–767px --- Вертикальная структура, бургер-меню

## Функционал и особенности реализации, интерактивность

1.  **БЭМ-методология**: Стили написаны по строгому стандарту БЭМ (Блок-Элемент-Модификатор), что исключает каскадные конфликты.
2.  **Компонентный CSS**: Архитектура шапки разбита на 5 файлов, что упрощает поддержку и масштабирование интерфейса.
3.  **Умные выпадающие списки (Dropdowns)**: Скрипт контролирует состояние селекторов валюты, языка и контактов. При открытии одного дропдауна остальные автоматически закрываются, также реализовано закрытие по клику вне области элемента.
4.  **Адаптивный слайдер отзывов**: На мобильных устройствах отображается по 1 отзыву в ряд, на планшетах и десктопах (`>= 768px`) сетка автоматически перестраивается на 2 отзыва. Реализована кнопка «далее...» для раскрытия длинных текстов.
5.  **Плавный скролл с авторасчетом**: Переход к блоку расписания по кнопке из Hero-секции рассчитывает высоту фиксированного хедера, предотвращая перекрытие заголовков.
6.  **Управление доступностью туров**: Карточки с классом `.tour-card--unavailable` визуально тускнеют, блокируют hover-эффекты и программно отключают возможность вызова алерта бронирования.
7.  **Бургер-меню**: Реализовано открытие/закрытие бургера с анимацией превращения в крестик.
8.  **Активная ссылка**: Подсветка текущего раздела при скролле.
9.  **Анимации появления**: Плавное появление и подъём заголовка в Hero-секции и карточек с расписанием экскурсий. Три анимированные галочки под кнопкой "Расписание" с разной прозрачностью и задержкой, создающие эффект волны. Иконки телефона в выпадающем списке Контактов и в футере мягко пульсируют, привлекая внимание.

## Инструкция по локальному развертыванию

Проект является статическим сайтом и не требует сборщиков (Webpack, Vite) или локального сервера Node.js для запуска.

1.  **Клонируйте репозиторий** или скачайте архив с проектом:
    ```bash
    git clone https://github.com/EleneKorniyuk/BrestTour.git
    ```
2.  **Важно (Кодировка файлы)**: Убедитесь, что все файлы стилей и скриптов сохранены в кодировке `UTF-8` (в некоторых версиях файлы имели кодировку Windows-1251, из-за чего комментарии могли отображаться некорректно).
3.  **Запустите проект**:
    *   Просто откройте файл `index.html` в любом современном браузере.
    *   *Рекомендуется*: Используйте расширение **Live Server** в VS Code для автоматической перезагрузки страницы при изменении кода.
